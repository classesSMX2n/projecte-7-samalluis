# P03: Repositori Personal

## 📝 Descripció
Aquest apartat del projecte se centra en la correcta organització i gestió del repositori de GitHub. El control de versions mitjançant **Git** és una eina fonamental, i aquest repositori serveix com a base oficial per al lliurament de totes les activitats del curs.

---

## 🚀 Introducció
L'objectiu principal és assegurar que el repositori estigui correctament estructurat i preparat per al lliurament final. S'han seguit les pautes establertes des de la tasca inicial per mantenir un flux de treball professional.

### 📂 Estructura del Repositori
Cada projecte o producte s'organitza en carpetes seguint un criteri de nomenclatura estricte:
* **Activitats:** `T01`, `T02`, `P01`, `P02`, etc.
* **Contingut de cada carpeta:**
    * `README.md`: Descripció de l'activitat o resum de l'enunciat.
    * `solucio.md`: Fitxer amb el codi, explicacions tècniques o enllaços corresponents.
    * `/img`: Carpeta local per a l'emmagatzematge de totes les imatges utilitzades en els documents.

---

## 🛠️ Requisits Tècnics
Per garantir la qualitat del lliurament, s'han aplicat els següents estàndards:
1.  **Format Markdown:** Tota la documentació està estructurada en seccions mitjançant sintaxi MarkDown professional.
2.  **Gestió d'Imatges:** Les imatges no estan disperses, sinó centralitzades en les seves respectives carpetes de recursos.
3.  **Nomenclatura:** Respecte absolut pel "Case Sensitivity" (exemple: `README.md` en majúscules).

---

## 📦 Lliurament
Segons les especificacions, el lliurament oficial es realitza exclusivament a través de:

> [!IMPORTANT]
> **Repositori assignat al GitHub Classroom.**
> No s'acceptaran lliuraments en repositoris externs o diferents de l'assignat oficialment.
